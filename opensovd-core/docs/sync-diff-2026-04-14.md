Target: make `opensovd-core` indistinguishable from upstream
`classic-diagnostic-adapter` (CDA) in every cross-cutting metadata / tooling /
workspace file, so future PRs upstream merge with zero stylistic friction.

## Second pass -- wholesale mirror (TDD)

Philosophy change (2026-04-14, second pass): user rejected "phase-gated" skips.
We now mirror CDA wholesale. Broken `cargo build` and broken CI are the
phase-by-phase punch list. TDD for infrastructure: the failures tell us what to
build next.

Only true divergences stay skipped (mirroring them would diverge, not mirror).
Advisory ignores that are scoped to deps we don't pull in are ADDED anyway --
if the advisory never applies, the ignore is harmless; if it does, we're ready.
